# Key管理优化设计文档

## 项目概述

优化AI平台原型页面中的key管理逻辑，实现项目级key与成员关联的完整解决方案，支持成员在Claude/Codex/Cursor等外部工具中使用分配的key。

## 业务需求

### 核心问题
1. 成员无法方便获取分配给他们的key去在外部工具使用
2. 缺少完整的成员key分配和权限管理流程
3. 无法有效追踪和监控成员key使用情况

### 目标用户
- **项目Owner**：管理项目key池，分配成员权限，监控使用情况
- **项目Member**：获取分配的key，查看个人使用统计
- **项目Viewer**：查看项目信息，无key使用权限

## 技术方案

### 架构选择
采用**完整架构方案**，参考OpenAI Enterprise、Azure OpenAI等主流企业AI平台的最佳实践：

#### 数据层优化
**新增核心表**：
```sql
-- 成员key分配关系表
project_member_key_assignments:
  - project_id, member_id, key_id (联合主键)
  - assigned_quota (分配的token配额)
  - role_level (Owner/Member/Viewer)
  - status (ACTIVE/SUSPENDED/EXPIRED)
  - assigned_by, assigned_at

-- key使用日志表
key_usage_logs:
  - key_id, member_id, usage_timestamp
  - token_consumed, api_endpoint
  - client_info (Claude/Cursor/Copilot标识)
  - cost_estimated
```

#### 业务逻辑层
**核心服务模块**：
- **KeyAssignmentService** - key分配/回收逻辑
- **UsageTrackingService** - token消耗追踪
- **PermissionService** - 三级权限控制
- **AlertService** - 配额告警

#### 展示层架构
**三级页面设计**：
- **管理员控制台** - key池管理、成员分配
- **成员个人中心** - key获取、使用统计
- **监控仪表盘** - 实时监控、成本分析

## 功能设计

### 1. 项目空间改进

#### 侧边栏结构调整
```
🏠 项目概览
👥 成员管理 (+key分配快捷操作)
🔑 Key管理中心 (新增)
🚀 服务管理
📊 监控运维 (+key使用监控)
```

#### Key管理中心页面
**四个核心区域**：

**A. Key池总览区**
- 项目总配额 vs 已分配配额
- 激活key数量状态
- 月度token消耗进度
- 费用预算使用情况

**B. 成员分配区**
- 成员列表表格展示
- 分配状态、配额、使用量
- 快捷分配/调整/暂停操作

**C. Key使用监控区**
- 实时使用情况面板
- Top活跃成员排行
- 异常告警提醒

**D. 快捷操作区**
- 批量分配功能
- 成员使用指南生成
- 使用报告导出

### 2. 成员个人视图

#### 项目内"我的Key"区域
- 分配key显示（脱敏处理）
- 一键复制配置代码
- 个人使用统计图表
- 外部工具配置指南

#### 权限分级功能
**三级权限模式**：
- **项目Owner**：完全管理权限
- **项目Member**：使用和查看权限
- **项目Viewer**：只读权限

### 3. 使用追踪监控

#### 实时监控功能
- Token消耗实时统计
- API调用频次追踪
- 成本预估和告警
- 异常使用检测

#### 报表和分析
- 成员使用趋势分析
- Key效率对比
- 成本优化建议
- 使用行为洞察
